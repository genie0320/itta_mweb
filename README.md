# ITTAKO 🗺️

사용자가 지정한 출발 위치를 기점으로 국내 소도시 관광지 및 축제를 탐색하고, KTX/전철/버스/도보가 연계된 최적의 멀티모달 대중교통 경로 안내 및 실제 네이버/카카오 지도 길찾기 앱 연동을 제공하는 데스크톱 대시보드 웹 서비스입니다.

---

## 🚀 주요 기능 (Key Features)

1. **데스크톱 4분면 대시보드 (4:2:2:2 분할)**
   - 가로 화면을 100% 가득 채워 좌측부터 `[지도(4) | 주변 관광지 목록(2) | 선택 관광지 상세(2) | 경로 안내(2)]` 영역으로 나눈 와이드 모니터용 통합 대시보드를 제공합니다.
   - 각 분할 영역은 독립적인 스크롤바가 개별 인가되어 탐색 편의성을 극대화했습니다.

2. **수동 주변검색 및 출발 핀 설정**
   - 지도의 임의 지점을 클릭하여 주황색 탐색 핀(출발지)을 꽂을 수 있으며, 클릭 조작 시에는 서버로의 자동 호출 트래픽이 전혀 발생하지 않습니다.
   - 상단의 **`[주변검색]`** 버튼을 클릭하는 시점에만 해당 핀 기준 주변 150km 내의 관광지를 조회하여 불필요한 서버 부하를 예방합니다.

3. **10개 단위 페이징 및 더보기 (`[더보기 (+10)]`)**
   - 대량의 관광지 마커 핀을 지도에 한 번에 그릴 때 생기는 브라우저 과부하(렉)를 방지하기 위해, 최초 10개만 목록과 핀으로 노출합니다.
   - 더보기 버튼 클릭 시 10개씩 점진적으로 추가 로드되어 쾌적한 렌더링 성능을 보장합니다.

4. **2단계 지연 경로 탐색 (Deferred Routing)**
   - 주변 관광지를 클릭하면 지도가 부드럽게 날아가고(FlyTo) 좌표 alert를 띄운 뒤, 3분면 상세 정보 패널만 업데이트합니다.
   - 3분면 상세 정보 하단의 **`[경로탐색 (대중교통)]`** 버튼을 직접 클릭하는 시점에만 경로 탐색 API를 조회함으로써 중복 트래픽 오버헤드를 막습니다.

5. **maasRP 멀티모달 상세 세로 타임라인**
   - ODsay의 멀티모달 대중교통 길찾기 API를 사용하여 KTX/SRT 열차, 시외/고속버스, 지하철, 시내버스 및 도보 이동(거리/시간 정보)을 촘촘히 엮어낸 단계별 세부 가이드 타임라인을 4분면에 출력합니다.
   - 검색 시점 기준 오전 10시로 시간을 자동 보정 고정하여 야간/새벽 시간 테스트 시 배차 끊김 에러를 사전에 예방합니다.

6. **수동 지정 출발 좌표 딥링크 동기화**
   - 네이버/카카오 지도 길찾기 시작 링크를 누르면, 실제 GPS 위치가 아닌 **내가 지도를 클릭하여 주황색 핀으로 꽂아둔 임시 출발지 좌표가 출발 위치로 자동 등록**되어 외부 지도 웹앱이 구동됩니다.

7. **로컬 축제 엑셀 데이터 연동 및 피클 고속 캐싱**
   - 한국관광공사 TourAPI 데이터에 더하여, 제공된 1,000여 건의 로컬 축제 데이터를 하버사인 거리 공식으로 계산해 통합 병합하여 지도에 초록색 핀으로 표출합니다.
   - 최초 1회 Excel 파싱 완료 후, 전처리 완료된 데이터프레임을 피클(`.pkl`) 캐시 파일로 로컬에 자동 보관하여 이후 조회 시 3초 ➔ 0.05초 만에 초고속 로딩을 지원합니다.

## 🛠️ 기술 스택 (Tech Stack)

- **Frontend**: React (Vite), Leaflet.js
- **Backend**: FastAPI (Python), Pandas (Data Processing), Uvicorn
- **Reverse Proxy**: Nginx (Auto-generating self-signed SSL for HTTPS connection)
- **Containerization**: Docker, Docker Compose

## 🏃 시작하기 (Getting Started)

### 1. 환경 설정 (.env)
프로젝트 루트 폴더에 `.env` 파일을 생성하고 아래 인증 키를 기입합니다.
```env
TOUR_API_KEY=발급받은_한국관광공사_투어API_인코딩인증키
ODSAY_API_KEY=발급받은_오디세이_API키
```

### 2. 컨테이너 가동 (Docker Run)
Docker 데스크톱이 켜져 있는 상태에서 프로젝트 루트 경로에서 아래 명령을 실행합니다.
```bash
docker compose up --build
```

### 3. 브라우저 접속
서버 가동이 정상적으로 끝나면 아래 주소 중 편한 경로로 접속합니다:
- **Vite 개발 서버 접속 (권장)**: `http://localhost:5173` (백엔드 프록시 라우팅 자동 지원)
- **Nginx HTTPS 프록시 접속**: `https://localhost` (브라우저 자체의 Geolocation 연동 보장)
