# [PoC] Tiny Sherpa 요구사항 정의서 (v0.1)

본 문서는 대도시에서 국내 소도시(로컬)로 이동하는 여행자를 위한 맞춤형 큐레이션 및 대중교통 가이드 서비스 'Tiny Sherpa'의 핵심 기능 검증(PoC)을 위한 최소 기능 구현 정의서입니다.

---

## 1. 서비스 개요 및 목표

- **서비스명**: Tiny Sherpa (타이니 셰르파)
- **형태**: 모바일 웹 애플리케이션 (Mobile Web)
- **목표**: 사용자의 위치를 기반으로 소도시의 매력적인 목적지(관광지/축제)를 제안하고, 대도시에서 해당 지점까지의 거시적 이동 경로를 제공한 후, 내비게이션은 메이저 지도 앱으로 매끄럽게 연동(Handoff)하는 흐름을 검증한다.

---

## 2. 핵심 아키텍처 및 데이터 흐름

```text
[사용자(모바일 웹)] ➔ HTML5 GPS 좌표 수집 ➔ 백엔드(FastAPI)
                       ├── 1. 한국관광공사 TourAPI (주변 관광지/축제 긁어오기)
                       └── 2. ODsay API (대도시-소도시 거점 대중교통 경로 요약)
[사용자(모바일 웹)] ➔ 외부 앱 핸드오프 버튼 ➔ 네이버맵/카카오맵 앱 구동 및 최종 목적지 자동 입력

```

---

## 3. 기능 요구사항 (Functional Requirements)

### F-1. 사용자 위치 좌표 식별

- **설명**: 서비스 진입 시 사용자의 현재 위경도 좌표를 확보한다.
- **상세**:
- 브라우저 내장 HTML5 Geolocation API를 사용하여 실시간 GPS 좌표를 획득한다.
- 모바일 웹 환경의 특성을 고려하여, 사용자 권한 거부 시 예외 처리(기본 좌표 설정 또는 지도에서 직접 선택 유도) 레이아웃을 제공한다.

### F-2. 위치 기반 로컬 콘텐츠 탐색 (API 연동)

- **설명**: 수집된 좌표를 기준으로 주변의 관광지 및 진행 중인 축제 정보를 리스트업한다.
- **상세**:
- **관광지 조회**: TourAPI 4.0 위치기반 정보 조회를 통해 반경 N km 이내의 관광지(`contentTypeId=12`)를 호출한다.
- **축제 조회**: 동일 반경 내 축제/행사(`contentTypeId=15`) 목록을 가져온다.
- **필터링**: 축제 데이터의 경우, 현재 날짜를 기준으로 이미 종료된 행사는 목록에서 제외하는 정제 로직을 백엔드에서 수행한다.

### F-3. 거시적 대중교통 경로 요약 (API 연동)

- **설명**: 현재 위치(출발지)에서 선택한 로컬 목적지까지의 대중교통 광역 이동 경로를 요약하여 보여준다.
- **상세**:
- ODsay 길찾기 API를 호출하여 최적 경로(기차, 고속버스, 시외버스 포함)를 탐색한다.
- 화면에는 복잡한 전체 타임라인 대신 **'총 소요 시간', '첫 승차 거점(기차역/터미널)', '최종 하차 로컬 거점'** 중심의 고밀도 요약 정보만 매핑한다.

### F-4. 메이저 지도 앱 핸드오프 (Handoff via Deep Link)

- **설명**: 웹 브라우저의 한계를 극복하기 위해, 실제 길 안내 및 라스트 마일 이동은 네이티브 지도 앱으로 이관한다.
- **상세**:
- URL 스킴(Custom URL Scheme)을 활용하여 사용자가 버튼 클릭 시 스마트폰에 설치된 '네이버 지도' 또는 '카카오맵' 앱을 강제 구동한다.
- 앱 구동 시 목적지의 위경도 좌표 및 장소명을 파라미터로 넘겨, 앱이 열림과 동시에 길찾기 화면이 즉시 프리셋되도록 구현한다.

---

## 4. 비기능 요구사항 및 기술 스택

- **Frontend**: React (Vite 기반, 모바일 웹 뷰 최적화 레이아웃)
- **Backend**: Python 3.10+ 기반 FastAPI (비동기 API 통신 처리)
- **인프라/보안**: 모바일 브라우저 GPS 하드웨어 권한 획득을 위해 **HTTPS** 프로토콜 적용 필수.

---

## 💡 전략적 통찰 및 검증 포인트 (Sherpa's Insight)

- **배차 간격의 리스크 관리**: 소도시/로컬 지역의 농어촌 버스는 하루 배차 횟수가 매우 적어 배차 정보가 누락되거나 틀어질 확률이 높습니다. PoC 단계 이후 프로덕션으로 고도화할 때는, 하차 터미널에서 목적지까지 대중교통 연결이 불확실할 경우 **"여기서부터는 지역 콜택시 호출을 권장합니다"** 등의 로컬 특화 안내 UI를 사전에 배치하는 것이 소도시 여행자에게 가장 강력한 '셰르파' 역할을 수행하는 핵심 밸류가 될 것입니다.
