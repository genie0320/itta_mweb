# ITTAKO Project Agent Rules & Guides

이 프로젝트(ITTAKO)에서 페어 프로그래밍이나 기능 개발을 수행하는 동료 에이전트들이 참고해야 할 핵심 개발 환경 가이드 및 트래픽/구조 제약 사항입니다.

---

## 🏃 1. 개발 서버 실행 포트 규격
- **포트 충돌 방지**: 로컬 타 프로젝트와의 충돌을 막기 위해 디폴트 포트 대역이 다르게 격리되어 있습니다.
  - **Vite Frontend Dev Server**: `http://localhost:3173`
  - **Nginx HTTPS Gateway (Reverse Proxy)**: `https://localhost:8443` (HTTP의 경우 `8080` 포트로 8443 리다이렉션)
- 컨테이너 기동 및 정지는 프로젝트 루트에서 `docker compose up -d --build` 및 `docker compose down` 을 사용하십시오.

---

## 🗺️ 2. 지연 대중교통 경로 탐색 (Deferred Routing) 규칙
- **API 트래픽 과부하 방지**: 한국관광공사 TourAPI 및 ODsay 대중교통 API의 불필요한 중복 조회를 최소화하기 위해 아래 유저플로우를 준수해야 합니다.
  1. 지도를 클릭하거나 드래그하는 조작 시점에는 **출발지 주황색 핀 좌표만 실시간 갱신**하며, API 조회를 자동 수행해서는 안 됩니다.
  2. 1분면 상단의 `[3시간 여행 (50km)]`, `[6시간 여행 (100km)]`, `[8시간 이상 (150km)]` 검색 버튼을 명시적으로 클릭하는 시점에만 해당 반경으로 주변 데이터(`explore`)를 획득하십시오.
  3. 목록 카드를 클릭할 때는 지도 FlyTo 이동과 위경도 alert 팝업만 작동시키고, 대중교통 경로 호출은 유보(Deferred)하여 3분면 상세 카드 정보만 채우십시오.
  4. 3분면 상세 정보 하단의 **`[경로탐색 (대중교통)]`** 버튼을 클릭하는 순간에 비로소 `/api/route` API를 찔러 4분면에 경로 데이터를 그리십시오.

---

## ⚡ 3. 대용량 엑셀 데이터 피클 캐싱
- **백엔드 고속화**: `backend/data/` 내부의 전국 축제 엑셀 데이터 파일은 최초 1회 기동 시 파싱 후 `backend/data/festivals.pkl` 피클 파일로 자동 직렬화 캐싱됩니다.
- 로컬 데이터 조회나 필터링 기능을 추가할 경우, 반드시 피클 파일을 먼저 읽어 들여 리소스 낭비와 3초 이상의 엑셀 파싱 딜레이를 방지하도록 설계하십시오.

---

## 🌗 4. 지도 한글화 및 실시간 테마 스위칭
- 지도는 국토교통부 브이월드(Vworld) 2D 공공 한글 지도셋을 사용합니다.
- 사용자가 라이트/다크 테마를 토글할 때, Leaflet의 `tileLayer` 인스턴스에 `setUrl()`을 호출하여 Base(라이트 한글 맵)와 Midnight(다크 한글 맵) 타일을 실시간으로 동적 전환해 주십시오. (지도 전체를 언마운트 후 재생성하는 무거운 제어는 피하십시오.)
